# 综合优化方案：打造高品质 HarmonyOS 社交应用 Demo

为了将当前的 Demo 提升为高品质的应用，我们将从**动画效果**、**UI 布局**、**机型适配**、**用户体验**和**功能完成度**五个维度进行全面优化。

## 核心优化目标
1.  **沉浸式体验**：实现首页到详情页的**一镜到底**（共享元素转场）动画。
2.  **功能补全**：完成**帖子详情页**（评论、点赞）和**聊天详情页**（气泡对话）。
3.  **多端适配**：确保在平板/折叠屏上展示**分栏布局**，而非简单的拉伸。
4.  **微交互**：添加点赞动画、骨架屏加载等细节。

---

## 📅 实施计划

### 第一阶段：视觉与动画升级 (Visual & Animation)
**目标**：让应用“动”起来，消除页面跳转的割裂感。

1.  **首页列表优化 (`HomeComponent`)**
    *   引入 **`geometryTransition` (共享元素转场)**：点击帖子图片时，图片平滑放大过渡到详情页，而非生硬跳转。
    *   添加 **骨架屏 (Skeleton Loading)**：在下拉刷新时展示闪烁的占位图，提升加载体验。
    *   优化 **瀑布流布局**：调整卡片间距和阴影，使其更符合 Material Design 或 HarmonyOS 设计规范。

2.  **帖子详情页重构 (`PostDetailPage`)**
    *   **全屏沉浸式设计**：顶部大图展示（支持手势关闭）。
    *   **内容补全**：展示完整的用户信息、正文、发布时间。
    *   **评论区实现**：添加模拟的评论列表，底部固定评论输入框。
    *   **点赞微交互**：点击爱心时触发缩放+弹跳动画。

### 第二阶段：核心功能补全 (Functional Completeness)
**目标**：让“消息”和“发现”模块不再是摆设。

1.  **聊天详情页开发 (`ChatDetailPage`)**
    *   新建聊天页面，实现气泡式对话布局（左侧对方，右侧自己）。
    *   支持发送文本消息，列表自动滚动到底部。
    *   从 `MessageComponent` 列表点击跳转至此页面。

2.  **发现页内容填充 (`DiscoverComponent`)**
    *   完善“话题广场”或“热门榜单”的子页面，展示不同维度的内容列表。

### 第三阶段：多端响应式适配 (Adaptive Layout)
**目标**：一套代码，多端优雅展示。

1.  **详情页分栏布局**
    *   **手机/折叠屏(折叠态)**：上下布局（上图下文）。
    *   **平板/折叠屏(展开态)**：左右分栏布局（左侧大图，右侧信息流和评论），充分利用宽屏优势。
    *   使用 `GridRow` 和 `GridCol` 或 `SideBarContainer` 实现。

2.  **输入法适配**
    *   确保聊天页和评论页的输入框在键盘弹出时自动顶起，不遮挡内容。

### 第四阶段：代码与性能优化 (Refactoring)
1.  **数据模型增强**：扩展 `Post` 和 `User` 模型，增加评论、聊天记录等 Mock 数据。
2.  **代码解耦**：将通用的 UI 组件（如头像组件、卡片组件）提取为独立的 `@Builder` 或 `Component`。

## ✅ 验收标准
1.  点击首页图片，图片能**平滑放大**进入详情页。
2.  在平板上打开详情页，能看到**左右分栏**的布局。
3.  点击消息列表，能进入聊天界面并**发送消息**。
4.  下拉刷新时能看到精致的**骨架屏动画**。
